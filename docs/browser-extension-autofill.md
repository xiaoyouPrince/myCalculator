# 浏览器扩展自动填报说明

`browser-extension` 是一个 Chrome / Edge / Arc Manifest V3 扩展，用于把 `myCalculator` 导出的极简 JSON 自动填入网页月度工时表单。

## 使用场景

App 负责记录本地工时数据，浏览器扩展负责把这些数据填到网页表单中。这样可以把“记录”和“填报”拆开：

- 日常只在 App 中维护上下班时间。
- 需要填网页时，从 App 导出极简 JSON。
- 在网页中打开扩展并执行自动填写。

## 数据格式

使用 App 左侧底部的 `导出极简 JSON`。导出的数据只包含工作记录，休假、自定义备注等非工作记录不会进入极简 JSON。

示例：

```json
[
  {
    "day": "2026-06-25",
    "startTime": "09:00",
    "endTime": "20:56"
  }
]
```

字段说明：

- `day`：日期，格式为 `yyyy-MM-dd`。
- `startTime`：上班时间，格式为 `HH:mm`。
- `endTime`：下班时间，格式为 `HH:mm`。

## 安装方式

- 打开 Chrome、Edge 或 Arc 的扩展管理页。
- 开启开发者模式。
- 选择“加载已解压的扩展程序”。
- 选择项目中的 `browser-extension` 目录。

## 使用流程

- 打开目标网页表单。
- 点击浏览器扩展图标。
- 导入 App 导出的极简 JSON。
- 选择填写模式。
- 点击“自动填写当前页”。
- 查看写入结果和月度汇总。

## 填写模式

扩展当前支持两种填写模式：

- `工作小时数`：按 `myCalculator` 当前工时申报规则计算后填写。
- `上下班时间段`：直接填写类似 `09:00-20:56` 的时间段。

默认小时精度为 1 位小数。填写成功后会显示本次导入数据的月度汇总：

- 记录天数
- 月工作时长
- 月工时申报
- 月加班时长
- 月有效加班

## 工时申报规则

扩展中的工时计算逻辑需要和 App 保持一致。当前规则为：

- 工作时长小于 10 小时：扣 1 小时。
- 工作时长等于 10 小时：不扣。
- 工作时长大于 10 小时：扣 2 小时。
- 加班时长为 `max(0, 工作小时 - 10)`。
- 有效加班小于 1 小时记 0；大于等于 1 小时按 0.5 小时向下取整。

如果后续 App 中 `WorkMetrics` 规则调整，浏览器扩展里的 `workMetrics` 也要同步修改。

## 匹配策略

扩展优先使用通用 DOM 扫描：

- 扫描表格行和单元格中的日期文本。
- 识别 `1` 到 `31` 或 `2026-06-25` 这类日期。
- 在日期附近查找可写的 `input` 或 `textarea`。
- 有匹配数据才填写，没有数据则跳过。

针对部分复杂网页，扩展也保留了专用匹配策略：

- 识别月度明细输入框。
- 按日期序号匹配 1 到 31 日。
- 在主世界执行写入逻辑，绕过部分页面框架对输入事件的限制。

## 诊断能力

如果自动填写失败，可以使用扩展中的诊断按钮。诊断结果会输出：

- 当前页面 URL。
- 推断出的年月。
- 导入记录日期。
- 候选日期数量。
- 可写输入框数量。
- 月度明细输入框列表。
- 页面表格文本样本。

诊断结果可以帮助判断是日期识别失败、输入框定位失败，还是页面框架拦截了写入事件。

## 维护提醒

- 目标网页 DOM 结构变化时，优先使用诊断功能确认输入框和日期锚点。
- App 的导出格式变化时，扩展的 `normalizeRecords` 需要同步更新。
- App 的工时计算规则变化时，扩展的 `workMetrics` 需要同步更新。
- 修改扩展后需要在浏览器扩展管理页重新加载。
